# Page Editor — Enrichment Design & Implementation Plan

This document specifies the target UX for each block editor, the data-model changes
required to support it, and a phased implementation roadmap.

---

## 1. Guiding Principles

- **Edit in place, preview inline.** Each block shows its authored content directly
  — no separate preview mode needed for simple types.
- **Tabs for source variants.** When a block has multiple source types (upload / URL /
  AI), surface them as tabs rather than a dropdown so all options are equally visible.
- **Collapse / expand.** Long blocks (article, translation, diagram) can be collapsed to
  a one-line summary to keep the page list scannable.
- **Drag handles.** Replace the ↑/↓ buttons with a left-side drag handle using
  `@dnd-kit/sortable` (already installed). Arrow buttons remain as keyboard alternative.
- **Unsaved-change indicator.** Each block that has been modified but not yet saved
  shows a small orange dot on its header. The page-level Save button is disabled when
  nothing has changed.
- **Shared infrastructure.** All block editors receive an optional `onAIGenerate`
  callback that opens a shared AI prompt modal (Phase 6 hook).

## 2. Block-by-Block Spec

### 2.1 `text`

**Current state:** bare Tiptap editor, no toolbar.

**Target:**
- Floating toolbar (appears on text selection): Bold, Italic, Underline, Code,
  Link, Heading levels H2/H3, Bullet list, Ordered list, Quote, Divider.
- Word-count footer (character count for short blocks).
- Placeholder text when empty: *"Start typing…"*
- Tiptap extensions to add: `@tiptap/extension-underline`, `@tiptap/extension-link`,
  `@tiptap/extension-character-count`.

**Data shape:** unchanged — `{ content: string }` (HTML).

---

### 2.2 `article`

**Current state:** identical to `text`.

**Target:**
- Same Tiptap toolbar as `text`, plus:
  - Heading H1 for article title section.
  - `@tiptap/extension-image` inline (insert image by URL or upload).
  - `@tiptap/extension-table` (insert basic tables).
  - Full-width layout (article blocks render wider than `text` blocks).
- Estimated read time shown in the footer (`Math.ceil(wordCount / 200)` min).
- Collapsible: click the block header to collapse to the first 80 chars.

**Data shape:** `{ content: string }` (HTML) — unchanged but can include `<img>` and
`<table>` tags generated by Tiptap.

---

### 2.3 `speech`

**Current state:** source-type dropdown + plain textarea for TTS text; upload shows only a storage key.

**Target:**

| Tab | Editor UI |
|-----|-----------|
| **TTS** | Textarea for text; voice selector (dropdown of available voices from `GET /api/v1/ai/voices` — Phase 6); "Generate Preview" button that calls `POST /api/v1/ai/generate-audio` and plays result via `<audio>` element in the browser. |
| **Upload** | `<input type="file" accept="audio/*">` with drag-drop zone; shows waveform thumbnail using the Web Audio API (`AnalyserNode`) after upload; plays back inline. |
| **Recorded** | `MediaRecorder`-based in-browser recorder: Record / Stop / Play / Re-record buttons; waveform visualised during recording; sends recorded blob via multipart. |
| **STT** | Textarea showing transcribed text (read-only once transcribed); "Transcribe" button that uploads audio and calls `POST /api/v1/ai/transcribe` (Phase 6); allows manual correction. |

**Conversation mode** (optional field `is_conversation: true`):
- When enabled, the data contains `turns: [{ role: "teacher"|"student", text: string }]`.
- UI shows a chat-like builder with Add Turn / Delete Turn / reorder rows.

**Data shape additions:**
```ts
interface SpeechBlockData {
  source_type: 'tts' | 'stt' | 'upload' | 'recorded'
  text?: string            // TTS source text
  voice?: string           // TTS voice id
  storage_key?: string     // Persisted audio file
  audio_url?: string       // External URL (fallback)
  duration_seconds?: number
  is_conversation?: boolean
  turns?: { role: 'teacher' | 'student'; text: string }[]
}
```

### 2.4 `flash_card`

**Current state:** plain textareas for front/back + interaction mode dropdown.

**Target:**
- **Live flip preview** rendered alongside the editor: a CSS `transform: rotateY(180deg)`
  card that flips on hover/click showing exactly what the student will see.
- **Rich text front/back** using a minimal Tiptap instance (bold, italic, code only).
- **Hint field** (`hint?: string`) — shown to students before they answer.
- **Interaction mode** expanded:
  - `flip` — student sees front, flips to reveal back.
  - `type` — student types the answer; back is the expected answer string.
  - `choice` — back text is one of N choices; add/remove distractor fields.
- **Distractors** (for `choice` mode): ordered list of strings; first is always the
  correct answer displayed in shuffled order to the student.

**Data shape additions:**
```ts
interface FlashCardBlockData {
  front: string
  back: string
  interaction_mode: 'flip' | 'type' | 'choice'
  hint?: string
  distractors?: string[]   // choice mode only
}
```

---

### 2.5 `image`

**Current state:** source-type dropdown; upload shows placeholder text.

**Target:**

| Tab | Editor UI |
|-----|-----------|
| **URL** | URL input + live preview thumbnail (`<img>` loaded from the URL). |
| **Upload** | Drag-drop zone + click-to-browse; shows image preview immediately (via `URL.createObjectURL`) before save; progress bar during multipart upload. |
| **AI Generate** | Prompt textarea; style selector (`photorealistic`, `illustration`, `diagram`, `watercolour`); "Generate" button (Phase 6 hook); shows spinner then rendered result. |

Additional fields visible regardless of source:
- **Alt text** — required for accessibility; editor shows warning badge if empty.
- **Caption** — optional text displayed below the image.
- **Display width** — `full`, `wide`, `medium`, `inline` radio group affecting
  CSS max-width in the student view.

**Data shape additions:**
```ts
interface ImageBlockData {
  source_type: 'url' | 'upload' | 'ai'
  external_url?: string
  storage_key?: string
  alt: string
  caption?: string
  display_width?: 'full' | 'wide' | 'medium' | 'inline'
  ai_prompt?: string
  ai_style?: string
}
```

---

### 2.6 `diagram`

**Current state:** raw JSON textarea — completely unusable in practice.

**Target:**
- Embed **Excalidraw** (`@excalidraw/excalidraw`) as an inline canvas inside the block.
  - The canvas is initially collapsed to a 320 × 200 px thumbnail; a "Edit diagram" button
    opens it full-screen in a modal overlay.
  - On save / close the `elements`, `appState`, and `files` fields are serialised back
    into `diagram_json`.
- Export button: PNG download of the current canvas.
- Fallback: if Excalidraw fails to load, fall back to the JSON textarea.

**Data shape additions:**
```ts
interface DiagramBlockData {
  diagram_json: {
    elements: unknown[]
    appState: Record<string, unknown>
    files?: Record<string, unknown>
  }
  thumbnail_key?: string   // Server-generated PNG stored in local storage
}
```

**Backend addition needed:** `POST /api/v1/blocks/:id/thumbnail` — accepts a PNG blob,
stores it, returns `thumbnail_key`. Called automatically when the modal closes.

---

### 2.7 `video`

**Current state:** source-type dropdown; embed shows URL input; upload shows placeholder text.

**Target:**

| Tab | Editor UI |
|-----|-----------|
| **Embed** | URL input that auto-detects YouTube / Vimeo / Loom and renders a live `<iframe>` preview at 16:9 aspect ratio. Paste detection (auto-populate on clipboard paste into the page). |
| **Upload** | Drag-drop zone accepting `.mp4`, `.webm`, `.mov`; HTML5 `<video>` preview before save; progress bar during multipart upload; shows file size and duration after upload. |
| **AI Generate** | Prompt textarea; duration selector (5 s / 10 s / 30 s); "Generate" button (Phase 6 hook). |

Additional fields:
- **Caption** — shown below the video.
- **Start time** (`start_seconds: number`) — for embed type, appended as `?t=N` to the URL.
- **Loop** (`loop: boolean`) — for upload type.
- **Autoplay** (`autoplay: boolean`) — for upload type; defaults to false.

**Data shape additions:**
```ts
interface VideoBlockData {
  source_type: 'embed' | 'upload' | 'ai'
  embed_url?: string
  storage_key?: string
  caption?: string
  start_seconds?: number
  loop?: boolean
  autoplay?: boolean
  ai_prompt?: string
  duration_hint?: '5s' | '10s' | '30s'
}
```

---

### 2.8 `translation`

**Current state:** flat list of language + text pairs.

**Target:**
- **Side-by-side layout:** source text (left column, read-only) next to each translation
  (right column, editable).
- **Language picker:** searchable dropdown (`<datalist>`) populated from ISO 639-1
  code list (bundled constant, no network call).
- **Pronunciation field** (`pronunciation?: string`) per translation — IPA or romanisation.
- **Audio per translation** (`audio_key?: string`) — small upload zone or TTS button
  next to each translation row; plays back inline.
- **Sentence segmentation view:** if source text contains multiple sentences (split on
  `.!?`), allow toggling into "sentence mode" where each sentence is shown with its
  own translation row to support phrase-by-phrase study.

**Data shape additions:**
```ts
interface TranslationEntry {
  language: string         // ISO 639-1 code, e.g. "ar"
  text: string
  pronunciation?: string
  audio_key?: string
}

interface TranslationBlockData {
  source_language: string
  source: string
  translations: TranslationEntry[]
  sentence_mode?: boolean
}
```

## 6. Backend Changes

### 6.1 Minor additions (no migration needed)

- `GET /api/v1/subjects/:id` already returns `cover_image_url`. No changes needed.
- The `blocks.data` column is already `jsonb` — all data-shape additions are purely
  additive and backwards compatible.

### 6.2 Diagram thumbnail endpoint

```
POST /api/v1/pages/:page_id/blocks/:block_id/thumbnail
Content-Type: multipart/form-data
  image: <PNG blob>

→ 200 { thumbnail_key: "abc123.png" }
```

- Implemented in `internal/content/handlers.go` alongside `replaceBlocks`.
- Stores via the existing `storage.Storage` interface.
- Updates `blocks.data` (merges `thumbnail_key` field).

### 6.3 Block-level data patch (nice-to-have)

Currently blocks can only be replaced in bulk. Add:

```
PATCH /api/v1/blocks/:id
Body: { data: {...} }   // partial merge into existing data
→ 200 Block
```

This avoids sending the full page's block array just to update one field (e.g., the
diagram thumbnail key or audio storage key after upload).

New DB query needed:
```sql
-- name: PatchBlockData :one
UPDATE blocks
SET data = data || $2::jsonb, updated_at = NOW()
WHERE id = $1
RETURNING *;
```

## 7. Implementation Plan

### Phase A — Editor infrastructure (prerequisite for all blocks)

1. Install Tiptap extensions listed in §5.
2. Build `BlockHeader` with drag handle (`@dnd-kit/sortable`), collapse toggle, type
   badge, unsaved indicator.
3. Replace `↑/↓` buttons in `ContentPage` / `BlockEditor` with `DndContext` +
   `SortableContext` drag-and-drop.
4. Build `TabBar`, `DropZone`, `InlineAudioPlayer` shared components.
5. Add `PATCH /api/v1/blocks/:id` backend endpoint + `patchBlockData` sqlc query.

### Phase B — Text & Article

1. Add floating toolbar to `RichTextEditor` (selection-based using Tiptap's `BubbleMenu`).
2. Add `Underline`, `Link`, `Placeholder`, `CharacterCount` extensions to `text` editor.
3. Add `Image`, `Table`, `Heading` extensions to `article` editor; full toolbar (fixed,
   not floating) using Tiptap's toolbar API.
4. Add word-count / estimated read time footer to `article`.
5. Add collapse-to-summary behaviour to `article`.

### Phase C — Image & Video

1. Build `DropZone` component; integrate into `ImageEditor` (upload tab).
2. Add live thumbnail preview for `url` tab via `<img>` with `onError` fallback.
3. Build `InlineVideoPreview` with auto-detect for YouTube/Vimeo/Loom embed URLs.
4. Add `DropZone` to `VideoEditor` (upload tab).
5. Wire `AIPromptModal` stub to `image` and `video` AI tabs (no real generation yet —
   just logs to console; Phase 6 will replace the stub).
6. Add `caption`, `display_width`, `start_seconds`, `loop`, `autoplay` fields to UIs.

### Phase D — Flash Card

1. Build `FlipCard` CSS component (perspective + `rotateY`).
2. Build `RichMiniEditor` (Tiptap with bold/italic/code, no toolbar).
3. Replace plain textareas in `FlashCardEditor` with `RichMiniEditor`.
4. Add `hint` field and `distractors` list (for `choice` mode).
5. Render live `FlipCard` preview to the right of the editor at all times.

### Phase E — Speech

1. Build `InlineAudioPlayer` (HTML5 `<audio>` + custom play/pause/seek bar).
2. TTS tab: voice selector + "Generate Preview" button stub.
3. Upload tab: `DropZone` accepting `audio/*`; plays back via `InlineAudioPlayer`
   using `URL.createObjectURL`.
4. Recorded tab: `MediaRecorder` wrapper; Record / Stop / Re-record; waveform
   visualisation using `AnalyserNode` + canvas.
5. Conversation mode toggle + turn list builder.

### Phase F — Translation

1. Build `LanguagePicker` (searchable select from bundled ISO 639-1 constant).
2. Add `pronunciation` field per row.
3. Add per-translation audio upload zone + `InlineAudioPlayer`.
4. Implement sentence-mode toggle (split source on `.!?` and create paired rows).

### Phase G — Diagram

1. Install `@excalidraw/excalidraw`.
2. Build `DiagramEditor` with lazy-loaded Excalidraw in a modal overlay.
3. Thumbnail generation: on modal close, call Excalidraw's `exportToBlob()` and POST
   to `POST /api/v1/pages/:page_id/blocks/:block_id/thumbnail`.
4. Show thumbnail image in collapsed block header.

## 9. Acceptance Criteria

| Block | Done when… |
|---|---|
| `text` | Floating toolbar works; links can be inserted; character count shown |
| `article` | Fixed toolbar with tables and inline images; read time footer; collapses |
| `speech` | All four tabs functional; recorder works in Chrome/Firefox/Safari; TTS preview plays |
| `flash_card` | Live flip preview updates as user types; `choice` mode shows distractors |
| `image` | Upload with preview, URL with live thumbnail, AI stub present; alt text warning |
| `diagram` | Excalidraw modal opens/closes; thumbnail shown in collapsed header |
| `video` | Embed auto-detects YouTube/Vimeo; upload with HTML5 preview; AI stub present |
| `translation` | Language picker; pronunciation field; per-translation audio; sentence mode |
| All | Drag-and-drop reorder; unsaved-change indicator; single Save persists everything |
